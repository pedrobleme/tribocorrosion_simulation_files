# **Tribocorrosion VUMAT Modeling**

### Setup and Execution Guide

---

## 1. Overview

This repository provides a complete numerical framework for the simulation of **tribocorrosion-driven material degradation** in structural components using the **finite element method**. The implementation integrates pre-processing scripts, a user-defined material subroutine (VUMAT), and post-processing utilities, enabling a **reproducible and physically consistent simulation workflow**.

The methodology is organized into four main stages:

1. Pre-processing of geometry-dependent element properties
2. Numerical simulation using Abaqus/Explicit and VUMAT
3. Post-processing of corrosion-induced mass loss
4. Visualization and analysis of long-term degradation trends

---

## 3. Phase I – Pre-Processing

### Generation of `element_properties.txt`

Before running any simulation using the VUMAT, an `element_properties.txt` file must be generated for the specific geometry being analyzed. This file contains element-wise parameters required by the corrosion and damage model.

This procedure is performed using the script `pre_processing.py`.

---

### 3.1 Extraction of Element Volumes (EVOL)

1. Prepare a preliminary Abaqus simulation using the target `.inp` file.
2. **Important:**
   Assign a standard built-in Abaqus material model.
   **Do not** assign the user-defined material (VUMAT) at this stage.
3. Run the simulation exclusively to extract the element volumes (`EVOL`).
4. Save the extracted EVOL values as a list to be used as input for the pre-processing script.

---

### 3.2 Execution of `pre_processing.py`

Run the script `pre_processing.py` from the main directory using the following inputs:

* The `.inp` file corresponding to the target geometry
* The EVOL list extracted in Step 3.1
* User-defined Weibull distribution parameters

**Outputs generated:**

* `element_properties.txt`
  Stored in the corresponding model folder
* A figure illustrating the applied Weibull statistical distribution with the generated values

---

## 4. Phase II – VUMAT Configuration and Pre-Run Checks

Once `element_properties.txt` has been generated, the simulation is nearly ready to run. However, the following checks **must** be performed before submitting the job.

---

### 4.1 Critical Check – Element Number Consistency

**Critical requirement:**
Always verify that the VUMAT reads the `element_properties.txt` file corresponding to the **same geometry** being simulated.

Using a properties file generated for a different model will result in a mismatch in the number of elements and will cause the simulation to fail.

---

### 4.2 Physical and Kinetic Parameters

Verify that all physical and kinetic parameters defined in the VUMAT are consistent with the intended simulation scenario:

* Corrosion rates
* Electrochemical kinetic parameters
* Mechanical wear coefficient
* Dynamic wear parameters

---

### 4.3 Geometry-Dependent Subroutine Adaptation

Each model folder contains a ready-to-run `.inp` file. However, the VUMAT must be manually adapted for the specific geometry being simulated.

In particular, the `block_surface` subroutine must be checked to ensure that all geometry-dependent parameters are correctly defined. This step is essential to:

* Restrict damage evolution to designated regions
* Apply appropriate boundary and interaction conditions
* Ensure numerical stability and physically meaningful results

---

## 5. Phase III – Simulation Execution

After completing the pre-processing stage and verifying all configuration checks, the simulation can be executed.

---

### 5.1 Software Environment

The simulations were performed using Abaqus coupled with the following compiler environment:

* Intel Parallel Studio XE 2015 (Composer Edition)
* Intel® 64 Compiler
* Microsoft Visual Studio 2013

---

### 5.2 Command-Line Execution

Navigate to the desired model directory and execute:

```bash
abaqus job=INP_FILE.inp user=VUMAT_FILE.for double=explicit
```

Double precision is required for explicit dynamic simulations.

---

## 6. Phase IV – Post-Processing

The `post_processing` directory contains scripts for extracting and analyzing corrosion-induced mass loss from the simulation results.

---

### 6.1 `calculate_mass_loss_odb.py`

This script performs the following tasks:

- Reads the Abaqus output database (`.odb`);
- Extracts corrosion-related state variables over time;
- Writes a CSV file containing the following columns:

where:

- **Frame**: Abaqus frame number;
- **Time**: Simulation time associated with the frame;
- **Sum_of_SDV2**: Spatially integrated value of the corrosion-related state variable (SDV2);
- **Total_Volume_Lost**: Cumulative volume loss due to corrosion, computed from the element-wise damage evolution.

**Important:**  
This script must be executed using the Abaqus Python interpreter:
```bash
abaqus python calculate_mass_loss_odb.py
```

---

### 6.2 `evolution_mass_loss.py`

This script reads the CSV file generated by `calculate_mass_loss_odb.py` and generates a plot of **corroded mass versus time**, enabling quantitative analysis of long-term corrosion and degradation trends.

---

## 7. Scope and Limitations

The computational framework provided in this repository is intended for **research and academic use**. The corrosion and tribocorrosion parameters implemented in the VUMAT are not universally calibrated and should be interpreted within the context of the modeling assumptions described in the associated thesis.

In particular:

* Weibull parameters are selected to represent predominantly uniform corrosion with controlled spatial heterogeneity
* Calibration against statistically representative pitting datasets is beyond the scope of the present implementation
* Users applying the framework to different materials or environments should reassess and recalibrate all physical and kinetic parameters accordingly
